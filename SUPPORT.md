# Support

Use the following channels so questions can be answered efficiently:

- **How-to questions:** open a GitHub Discussion when the repository has Discussions enabled.
- **Bug reports:** use the bug report issue form and include reproduction steps, logs, versions, and screenshots where useful.
- **Feature proposals:** use the feature request form or start a discussion for larger product changes.
- **Security reports:** follow [SECURITY.md](SECURITY.md); do not post sensitive details publicly.

Before asking for help, search the documentation, existing issues, and discussions. A minimal reproduction is the fastest way to get a useful answer.
