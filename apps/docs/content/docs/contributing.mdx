# Contributing

Read the repository-level [`CONTRIBUTING.md`](https://github.com/socializ-co/socializ/blob/main/CONTRIBUTING.md) before opening a pull request.

Contributions should be focused, tested, documented when appropriate, and aligned with the project's domain boundaries. Use the issue forms for bugs and feature requests, and include verification commands in every pull request.

For security reports, follow the private process in [`SECURITY.md`](https://github.com/socializ-co/socializ/blob/main/SECURITY.md). Never include secrets or private data in an issue, pull request, log, or screenshot.
