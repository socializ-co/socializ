# Governance

## Project roles

- **Maintainers** review changes, protect the project's direction, manage releases, and respond to security or conduct reports.
- **Contributors** improve the code, documentation, tests, design, and community through issues and pull requests.
- **Community members** provide feedback, reproduce bugs, answer questions, and help shape proposals.

## Decision making

Small, reversible changes can be proposed directly in a pull request. Changes that affect public APIs, domain boundaries, data models, security, or deployment should start with an issue or architecture decision record.

Maintainers seek consensus through review. When consensus is not possible, the maintainers make the final decision and document the reasoning in the issue, pull request, or ADR.

## Becoming a maintainer

Maintainer invitations are based on sustained, high-quality contributions, sound judgment, respectful collaboration, and an understanding of the project's architecture and users. The existing maintainers make the decision and document the change.

## Changes to governance

Governance changes require a reviewed pull request and approval from the active maintainers.
